# kubebuilder Operator — DevTask Controller

## Project layout

```
operator/
├── api/v1alpha1/
│   ├── devtask_types.go         — CRD spec, status, phase constants
│   └── zz_generated.deepcopy.go — auto-generated, never edit
├── cmd/main.go                  — entry point, registers controller
├── config/
│   ├── crd/bases/               — generated CRD YAML
│   ├── rbac/                    — generated RBAC manifests
│   └── samples/                 — example DevTask YAML
└── internal/controller/
    ├── devtask_controller.go    — Reconcile function and state machine
    ├── namespace.go             — namespace create/delete helpers
    ├── networkpolicy.go         — NetworkPolicy for sandbox egress
    ├── pod.go                   — agentPod / agentPodResume builders
    ├── secrets.go               — per-task Secret helpers
    └── github_poller.go         — polls GitHub, creates DevTask CRs
```

## Development workflow

### After changing CRD types (devtask_types.go)

```bash
cd operator/
make generate   # regenerates zz_generated.deepcopy.go
make manifests  # updates config/crd/bases/ YAML
make install    # applies updated CRDs to the cluster
```

Always run `make generate && make manifests` before `make install` when types change.

### Run the operator locally

```bash
cd operator/
export GITHUB_TOKEN=<token>
export ANTHROPIC_API_KEY=<key>
make run
```

The operator connects to whichever kubeconfig context is current (`kubectl config current-context`). For development this should be `k3d-slaktforskning-poc`.

### Build the operator image

```bash
cd operator/
IMG=slaktforskning-registry:5000/operator:dev make docker-build docker-push
```

### Apply a test DevTask

```bash
# Edit issueNumber to match a real open issue
kubectl apply -f config/samples/devtask-sample.yaml

# Watch it progress
kubectl get devtask -n devpipeline-system --watch

# Watch the agent pod in the sandbox namespace
kubectl logs -n devtask-<N> agent --follow
```

### Delete a DevTask and its namespace

```bash
kubectl delete devtask slaktforskning-<N> -n devpipeline-system
kubectl delete namespace devtask-<N>  # if not already cleaned up by the controller
```

## State machine phases

| Phase | Meaning |
|---|---|
| `""` (empty) | Just created; Reconcile will create namespace, NetworkPolicy, Secret, pod |
| `Building` | envbuilder building the devcontainer image |
| `Running` | `claude -p` is running |
| `AwaitingReview` | Agent exited 0, PR is open |
| `BlockedOnClarification` | Agent exited 2 with `/clarification:` comment; pod+NS deleted |
| `Failed` | Agent pod failed non-zero (not clarification) |
| `Completed` | PR merged or closed; namespace deleted |

## Required RBAC for the operator

The operator needs these cluster permissions (generated by kubebuilder markers on the Reconcile function):

```go
// +kubebuilder:rbac:groups=devpipeline.local,resources=devtasks,verbs=get;list;watch;create;update;patch;delete
// +kubebuilder:rbac:groups=devpipeline.local,resources=devtasks/status,verbs=get;update;patch
// +kubebuilder:rbac:groups="",resources=namespaces,verbs=get;list;watch;create;delete
// +kubebuilder:rbac:groups="",resources=pods,verbs=get;list;watch;create;delete
// +kubebuilder:rbac:groups="",resources=secrets,verbs=get;list;watch;create;delete
// +kubebuilder:rbac:groups=networking.k8s.io,resources=networkpolicies,verbs=get;list;watch;create;delete
```

Add these above the Reconcile function; `make manifests` will regenerate `config/rbac/role.yaml`.

## Troubleshooting

**CRD not found error:** Run `make install` to apply the latest CRD to the cluster.

**"cannot create resource: namespaces" error:** The operator's service account lacks namespace creation permission. Add the RBAC marker above and run `make manifests && kubectl apply -f config/rbac/`.

**Pod stuck in `ContainerCreating`:** The envbuilder image may be pulling from `ghcr.io`. Check pod events:
```bash
kubectl describe pod -n devtask-<N> agent
```

**Reconcile loop not running:** The operator's manager needs the CRD to be installed. If the CRD is missing, the informer fails to list/watch. Check operator logs.
